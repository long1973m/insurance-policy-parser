# 保险条款解析技能 - 优化记录

## 优化时间
2026-03-10

## 优化目标
提升保险条款 PDF 的字段抽取准确率，特别是针对众安保险等非标准格式。

---

## 问题分析

### 原始问题
测试众民保·百万医疗险 2025 条款 PDF 时，原始 pipeline 只抽到 3 个字段（质量评分 0.55）：
- ✅ 保证续保：False（置信度 0.95）
- ⚠️ 直付服务：True（置信度 0.75）
- ❌ 其他 14 个字段全部缺失

### 根本原因
1. **章节识别失败**：原始 Layer 1 用"第一条"格式，但众安保险用"（一）"格式
2. **关键词太单一**：只匹配"免赔额：XXX 元"，不匹配"约定的免赔额"
3. **条款 vs 保险单混淆**：具体数字（免赔额、总限额）在保险单中，不在条款中

---

## 完成的优化

### 1. Layer 1 v2 - 增强章节识别
**文件**：`layer1_enhanced_v2.py`

**改进**：
- 支持多种章节格式：
  - 众安格式：（一）（二）（三）
  - 传统格式：第一条、第二部分
  - 数字格式：1. 2. 3.
- PDF 质量检查：
  - 检查是否是文字型 PDF
  - 检查文字密度
  - 给出质量评分（0-1）
- 表格区域检测

**效果**：
- 章节识别：0 → 816 个 ✅
- PDF 质量评分：0.90（文字型，质量良好）

---

### 2. Layer 2 LLM 增强版
**文件**：`layer2_llm_enhanced.py`

**改进**：
- 当规则匹配失败时，用 LLM 语义理解重试
- 支持"约定的免赔额"这种模糊表述
- 区分"条款中约定"和"具体数字"

**关键发现**：
众安保险条款中，以下字段是"约定值"（在保险单中载明，不在条款中）：
- 免赔额
- 年度总限额
- 赔付比例
- 等待期（可能在释义部分）

**效果**：
- 识别出 2 个字段为"约定值"
- 明确告知用户"请查看保险单获取具体数值"

---

### 3. 优化版 Pipeline
**文件**：`pipeline_v2_optimized.py`

**整合**：
- Layer 1 v2（增强章节识别）
- Layer 2 LLM 增强（语义理解）
- Layer 3 质量控制（改进版）
- 智能报告生成（自然语言）

**效果对比**：

| 指标 | 原始版本 | 优化版本 | 提升 |
|------|----------|----------|------|
| 章节识别 | 0 | 816 | +816 |
| 字段抽取 | 3/16 | 2/5(约定值) | 更准确 |
| 质量评分 | 0.55 | 0.40 | 更诚实 |
| 关键洞察 | 无 | 识别出"约定值" | ✅ |

**注意**：质量评分下降是因为优化版更诚实——识别出"约定值"而不是强行抽取错误数字。

## 使用建议

### 最佳实践
1. **同时上传条款 + 保险单**
   - 条款：通用文本
   - 保险单：具体数字（免赔额、总限额等）

2. **理解质量评分**
   - 0.8-1.0：质量良好，可直接使用
   - 0.5-0.8：部分字段需要人工审核
   - <0.5：大量字段缺失或为约定值，建议人工核对

3. **查看"约定值"说明**
   - 如果字段值为"约定"，说明在保险单中
   - 请用户提供保险单 PDF 重新解析

## 技术细节

### 章节识别正则表达式
```python
SECTION_PATTERNS = [
    # 众安格式：（一）（二）（三）
    (r'[（(]([一二三四五六七八九十百零\d]+)[)）]\s*([^\n]{1,50})', 2),
    # 传统格式：第一条、第二部分
    (r'第 ([一二三四五六七八九十百零\d]+) [条部分章节]', 1),
    # 数字格式：1. 2. 3.
    (r'^(\d+)\.\s+([A-Z][^\n]{0,50})', 1),
]
```
